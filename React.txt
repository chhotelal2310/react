A Higher-Order Component (HOC) is a function that take a component as an argument and return a new enhanced component with additional functionality.
Syntax:const EnhancedComponent = higherOrderComponent(OriginalComponent);

Advantages of HOC:
1.Reuse code across multiple components.
2.Reduce code duplication.
3.Keep components clean and simple.
4.Easy to maintain.
5.Separate UI from business logic.

Disadvantages of HOC:
1.Nested Wrappers
Ex. withAuth(withTheme(withLoading(Profile)));
2.Harder to debug.
3.Props Collision



Example:
const User = ({ users }) => {
  return (
    <ul>
      {users.map((user) => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  );
};

const withLoading = (Component) => {
  return function ({ loading, ...props }) {
    if (loading) {
      return <h2>Loading...</h2>;
    }
    return <Component {...props} />;
  };
};

const UserWithLoading = withLoading(User);

export default UserWithLoading;

Use:
<UserWithLoading loading={true} users={[{ id: 1, name: "Rahul" }]} />



What is children Prop in React?
The children prop is a special prop in React used to pass content (text, JSX, components, elements) between the opening and closing tags of a component.

Syntax: <Component>Content Here</Component>;



A Fragment is a React component used to group multiple JSX elements without adding an extra HTML element to the DOM.
Example:1 Shorthand Fragment (Iternaly it is wroked Named Fragment)
<>
  <h1>Hello</h1>
  <p>React</p>
</>

Example:2 Named Fragment
<React.Fragment>
  <h1>Hello</h1>
  <p>React</p>
</React.Fragment>

Note: Shorthand Fragment syntax cannot receive props like key.

/*********************************************** Lazy Loading *******************************/
Example with routing:
import React, { Suspense } from "react";
import { BrowserRouter, Routes, Route } from "react-router-dom";

const Home = React.lazy(() => import("./pages/Home"));
const About = React.lazy(() => import("./pages/About"));
const Contact = React.lazy(() => import("./pages/Contact"));

function App() {
  return (
    <BrowserRouter>
      <Suspense fallback={<h2>Loading page...</h2>}>
        <Routes>
          <Route path="/" element={<Home />} />
          <Route path="/about" element={<About />} />
          <Route path="/contact" element={<Contact />} />
        </Routes>
      </Suspense>
    </BrowserRouter>
  );
}

export default App;


Benefits of React.lazy()
1. Faster initial load
2. Better performance
3. Smaller bundle size
4. Better user experience

Limitations of React.lazy()
1. It works only for default export
const About = React.lazy(() => import("./About"));
About.js
export default About;

Not directly for named export:
export const About = () => {};

2. It must be used inside Suspense: If not, React will throw an error.
3. It is for components only: You cannot use React.lazy() for normal functions, variables, or hooks.
                              It is only for React components.



How to use named export with React.lazy()
export const About = () => {
  return <h1>About Page</h1>;
};
const About = React.lazy(() =>
  import("./About").then((module) => ({
    default: module.About,
  }))
);



What is code splitting?
Code splitting means dividing your JavaScript into smaller parts so that only the needed part is loaded when required. React.lazy() helps with this.

What happens if Suspense is not used?
React will throw an error because it does not know what to show while the lazy component is loading.
